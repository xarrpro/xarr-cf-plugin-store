# CF Plugin Store — Node CLI 实现计划(计划 B)

> 状态:已实现并通过测试(15 个 CLI 用例)。本文档记录 CLI 的设计与任务分解。

**Goal:** 提供 npm-publish 风格的 Node CLI `xplugin`,让作者一键创建、打包、签名并发布插件到 Worker 服务端。

**Architecture:** 纯 Node(ESM)。fflate 打包 zip;Node 内置 crypto 做 sha256 与 Ed25519;Node fetch 上传。与服务端契约对齐:raw body 上传 + `X-Package-Sha256` + 可选 `X-Package-Signature`。

**Tech Stack:** TypeScript, Node crypto, fflate, vitest(node 环境,独立配置)。

设计依据:`docs/superpowers/specs/2026-06-07-cf-plugin-store-design.md` §9。

## 文件结构

```
cli/
├─ src/
│  ├─ manifest.ts   # 读取并校验 manifest.json(必填 name/title/version/type)
│  ├─ pack.ts       # 打包 zip(manifest/plugin.lua/index.html/README)+ sha256
│  ├─ sign.ts       # Ed25519 密钥生成 / 对 sha256 hex 签名 / 本地验签
│  ├─ config.ts     # 配置加载(~/.xplugin/config.json + 环境变量覆盖)
│  ├─ init.ts       # 生成插件骨架 + 密钥对
│  ├─ publish.ts    # pack→签名→(创建插件)→raw body 上传;--channel/--dry-run
│  └─ cli.ts        # 命令分发(init/pack/publish)+ flag 解析
├─ bin/xplugin.mjs  # 可执行入口
├─ test/cli.test.ts # 15 个用例
└─ vitest.config.ts # node 环境,独立于 worker 的 workers-pool 配置
```

测试隔离:根 `vitest.config.ts` 加 `include:["test/**/*.test.ts"]` + `exclude:["cli/**"]`;CLI 用 `cli/vitest.config.ts`(node 环境);根 `package.json` 加 `test:cli` 脚本。

## 命令

### `xplugin init [name]`
在目标目录生成 `manifest.json`、`plugin.lua`、`README.md`,并生成 Ed25519 密钥对(`ed25519_private.pem` / `ed25519_public.pem`),打印 raw 公钥 hex(供配置到服务端验签)。

### `xplugin pack`
读取当前目录 → 校验 manifest 必填字段 → fflate 打包 zip → 计算 sha256 → 写出 `{name}-{version}.zip`。

### `xplugin publish [--channel=beta] [--dry-run]`
pack → 若配置了签名私钥则对 sha256(hex)做 Ed25519 签名 → 读配置 → 先 `POST /admin/plugins` 创建插件(已存在 409 则忽略)→ `POST /admin/plugins/{name}/releases` 以 raw body 上传 zip,带 `X-Package-Sha256` 与可选 `X-Package-Signature`,`--channel` 非 stable 时加 `?channel=`。`--dry-run` 仅本地打包+签名,不发任何请求。

配置来源:环境变量 `XPLUGIN_BASE_URL/XPLUGIN_GATEWAY_UUID/XPLUGIN_TOKEN/XPLUGIN_SIGN_KEY` 优先,其次 `~/.xplugin/config.json`。

## 任务分解(TDD,已完成)

- Task 1 manifest:`loadManifest` 读取+必填校验。测试:合法解析 / 缺文件 / 缺字段。
- Task 2 pack:`pack` 打包+sha256。测试:产物可解压含 manifest / sha256 一致 / 缺 manifest 抛错。
- Task 3 sign:`generateKeypair`/`signHex`/`verifyHex`。测试:签名可验签通过、篡改失败;publicHex 为 64 hex。
- Task 4 config:`loadConfig`/`assertPublishConfig`。测试:环境变量覆盖 / 缺字段抛错。
- Task 5 init:`init` 生成骨架+密钥。测试:文件齐全、manifest 可被 loadManifest 读取。
- Task 6 publish:`publish`。测试:dry-run 不发请求 / 正常流程断言 URL+headers+raw body / 409 继续 / channel=beta 带 query。
- Task 7 cli 入口:`run` 分发 + `bin/xplugin.mjs`。
